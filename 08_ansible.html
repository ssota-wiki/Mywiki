1. 概要 & ディレクトリ構成

4. カスタムコレクション（Alaxala用）

cisco.ios プラグインが Alaxala と非互換（ターミナルパラメータ設定エラー）だったため、local.network.network を自作しました。

  • ページャ制御を自動で行わない（各コマンドで set terminal pager disable を明示指定）
  • on_open_shell() を空実装にして接続時の余計な初期化コマンドを送らない
  • CISCOライクなプロンプト（#）に対応
group_vars/alaxala.yml
ansible_connection: ansible.netcommon.network_cli
ansible_network_os: local.network.network

5. インベントリ設定

group_vars（接続情報）

group_vars/juniper.yml
ansible_connection: ansible.netcommon.network_cli
ansible_network_os: junipernetworks.junos.junos
ansible_command_timeout: 30
group_vars/apresia.yml
ansible_connection: ansible.netcommon.network_cli
ansible_network_os: apresia.aeosnp.aeosnp
ansible_network_cli_ssh_type: libssh
ansible_command_timeout: 30
ansible_libssh_publickey_algorithms: "ssh-rsa,rsa-sha2-256,rsa-sha2-512"
ansible_libssh_hostkeys: "ssh-rsa,ssh-dss"
ansible_libssh_key_exchange_algorithms: "diffie-hellman-group14-sha1,diffie-hellman-group1-sha1,curve25519-sha256"
ansible_libssh_host_key_checking: false

host_vars（個別コマンド投入）

host_vars/JUN06.yml
target_config_cmds:
  - "configure"
  - "set system host-name JUN06-TEST"
  - "commit"

6. 使い方

bash
# 全台実行
ap -i inventory/hosts.ini playbooks/changeConfig.yml

# 特定機器のみ
ap -i inventory/hosts.ini playbooks/changeConfig.yml -l JUN06
ap -i inventory/hosts.ini playbooks/changeConfig.yml -l apresia

# SSH疎通確認のみ
ap -i inventory/hosts.ini playbooks/checkLogin.yml

7. Playbook 処理フロー

1
Connection Check — ポート22 疎通確認
2
Set Fact — target_config_cmds を final_apply_cmds にセット
3
Pre-Check — 各機器の getLog.yml を実行（pre_res に格納）
4
Apply Configuration — 各機器の apply.yml を実行（コマンド投入）
5
Post-Check — 各機器の getLog.yml を実行（post_res に格納）
6
Save Log — common/outputDiff.yml でログ出力（Pre/Post 差分比較）

8. ログ出力

ログは logs/ に {ホスト名}_{タイムスタンプ}.log で保存されます。

log format (outputDiff)

[STATUS]

9. デバッグ & よくあるエラー

bash
# 詳細ログ
ap -i inventory/hosts.ini playbooks/changeConfig.yml -l JUN06 -vvv

# 接続タイプの確認
ap -i inventory/hosts.ini playbooks/changeConfig.yml -l APR09 -vvvv 2>&1 | grep "ssh type"

# dry-run（接続確認のみ）
ap -i inventory/hosts.ini playbooks/changeConfig.yml --check
エラー原因対処
kex error: no match for method server host key algoSSH鍵アルゴリズム不一致ansible_libssh_hostkeys / publickey_algorithms を設定
unable to set terminal parametersターミナルプラグイン非互換local.network.network カスタムプラグインを使用
'post_res' is undefinedPost-Check の set_fact が未実行getLog.yml に set_fact が含まれているか確認
command timeout triggeredコマンドタイムアウトansible_command_timeout を増やす
group_vars が読まれないインベントリパスが違うansible.cfg のある main/ ディレクトリから実行する
