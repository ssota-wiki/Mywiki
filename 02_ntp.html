ℹ 前提 — このページの手順は AlmaLinux 9 セットアップ完了後に実施してください。パッケージ導入・Firewalld/SELinux 無効化は 01. AlmaLinux 9 を参照。

1. Timezone 設定

bash
timedatectl set-timezone Asia/Tokyo
bash – 確認
timedatectl

2. NTP クライアント

パッケージのインストール

bash
dnf -y update && dnf -y install chrony

変数を設定

bash
NTP_SV_1="192.168.1.1"
NTP_SV_2="192.168.1.2"
SERVER_TYPE="server"           # pool | server

設定ファイルのバックアップ

bash
cp -p /etc/chrony.conf /etc/chrony.conf.org

chrony.conf の設定

bash
sed -i -E '/^(server|pool)/s/^/#/' /etc/chrony.conf
sed -i "/$(grep -E "^#(server|pool)" /etc/chrony.conf | tail -n 1)/a ${SERVER_TYPE} ${NTP_SV_1} iburst" /etc/chrony.conf
test -z "${NTP_SV_2}" || sed -i "/${NTP_SV_1}/a ${SERVER_TYPE} ${NTP_SV_2} iburst" /etc/chrony.conf

サービス起動

bash
systemctl enable chronyd.service && systemctl restart chronyd.service

動作確認

bash
chronyc -n sources -v && echo && date
output

3. NTP サーバー

変数を設定

bash
NTP_SV_1="ntp.nict.jp"
NTP_SV_2="2.almalinux.pool.ntp.org"
NTP_CLIENT="192.168.0.0/16"
SERVER_TYPE="pool"           # pool | server

chrony.conf の設定（サーバー用・クライアント許可追加）

bash
sed -i -E '/^(server|pool)/s/^/#/' /etc/chrony.conf
sed -i "/$(grep -E "^#(server|pool)" /etc/chrony.conf | tail -n 1)/a ${SERVER_TYPE} ${NTP_SV_1} iburst" /etc/chrony.conf
test -z "${NTP_SV_2}" || sed -i "/${NTP_SV_1}/a ${SERVER_TYPE} ${NTP_SV_2} iburst" /etc/chrony.conf
test -z ${NTP_CLIENT} || sed -i "/$(grep -E "^#allow" /etc/chrony.conf | tail -n 1)/a allow ${NTP_CLIENT}" /etc/chrony.conf

サービス起動 & ポート確認

bash
systemctl enable chronyd.service && systemctl restart chronyd.service
lsof -i:123

クライアント接続確認

bash
chronyc -n clients -v

4. デバッグコマンド

コマンド内容
chronyc -n tracking同期サーバーの情報表示
chronyc makestep時刻を即時強制修正
chronyc refreshFQDN の再解決
chronyc reselect同期源の再選択
