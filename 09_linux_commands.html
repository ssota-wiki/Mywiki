🖥 nmtui — ネットワーク設定（擬似GUIツール）

// CUI 上で動く擬似 GUI

nmtui とは？

nmtui（Network Manager Text User Interface）は、ターミナル上でマウス感覚で操作できる擬似 GUI ツールです。 コマンドを暗記しなくても、矢印キー・Tab キー・Enter キーだけで IP アドレスや DNS の設定・変更ができます。 SSH 接続先で IP を変えたいとき、設定ファイルを直接編集せずに済むので非常に便利です。

起動コマンド説明備考
nmtui
メニュー画面を開く3つのメニューが表示される
nmtui edit
接続設定の編集画面を直接開くIP / DNS / Gateway の変更
nmtui connect
接続の有効化・無効化画面を開くNIC の on/off 切り替え
nmtui hostname
ホスト名の変更画面を開く再起動なしで即時反映

操作方法：↑↓ で項目移動 → Tab で次のフィールドへ → Enter で決定 → Esc でキャンセル・戻る

🔍 System Info

コマンド用途補足
cat /etc/os-release
OS バージョン確認AlmaLinux のバージョンを確認
uname -r
カーネルバージョン-a でフル情報
hostnamectl
ホスト名・OS 情報set-hostname で変更可
uptime
稼働時間・負荷確認load average の目安はコア数以下が正常
lscpu
CPU 情報（コア数・型番）nproc でコア数だけ表示
free -h
メモリ使用量-h で人が読みやすい単位
timedatectl
日時・タイムゾーン確認set-timezone Asia/Tokyo で変更

⚙️ Process

コマンド用途補足
ps aux --sort=-%cpu | head -20
CPU 使用率 TOP20--sort=-%mem でメモリ順
pgrep -la nginx
プロセス名で検索PID と名前を表示
kill -9 PID
プロセス強制終了pkill -f 名前 でも可
top -d 2
リアルタイム監視htop が見やすい（dnf install -y htop）

🌐 Network

コマンド用途補足
ip a
IPアドレス一覧ip -br a でコンパクト表示
ip route show default
デフォルトGW確認ip route で全ルート表示
ss -tlnp
LISTEN 中のポート確認旧 netstat の代替コマンド
ping -c 4 8.8.8.8
疎通確認traceroute でルート確認
dig example.com
DNS 名前解決確認@8.8.8.8 で DNS サーバー指定
firewall-cmd --list-all
FW 設定確認--add-port=8080/tcp --permanent で追加
tcpdump -i eth0 port 22 -n
パケットキャプチャ-w /tmp/cap.pcap でファイル保存

💾 Disk / Storage

コマンド用途補足
df -hT | grep -v tmpfs
ディスク使用量確認T でファイルシステム種別も表示
du -sh /* 2>/dev/null | sort -rh | head -10
容量の大きいディレクトリ TOP10容量逼迫時の調査に便利
lsblk
ブロックデバイス一覧ディスクとパーティション構成を確認
iostat -xz 1 5
ディスク I/O 確認1秒間隔×5回 (sysstat が必要)

📋 Log / Search

コマンド用途補足
journalctl -xe
systemd ログ（エラー中心）-u サービス名 で絞り込み
journalctl -u radiusd -f
特定サービスをリアルタイム監視-f で tail -f 相当
journalctl -p err -b
今回起動のエラーだけ表示-b 0 で今回起動分
tail -f /var/log/messages
ログのリアルタイム監視| grep failed で絞り込み可
grep -A 3 -B 3 "ERROR" /var/log/messages
エラー前後 3 行も表示-A: 後ろ行 / -B: 前の行
grep -rn "keyword" /etc/
設定ファイル内を再帰検索-n で行番号も表示
find /etc -name "*.conf" -mtime -1
直近 1 日に変更された設定ファイル-mmin -60 で 60 分以内

🔧 Service（systemd）

コマンド用途補足
systemctl start SERVICE
サービス起動stop / restart / reload も同様
systemctl enable --now SERVICE
自動起動 ON + 即時起動--now を付けると今すぐ起動も兼ねる
systemctl status SERVICE --no-pager
サービス状態確認--no-pager で画面に全部出す
systemctl list-units --type=service --state=failed
失敗しているサービス一覧トラブル調査の最初に確認

📁 File Ops

コマンド用途補足
tar czf backup.tar.gz /target/
圧縮アーカイブ作成xzf で解凍 / tzf で中身確認
scp file.txt user@192.168.1.10:/tmp/
ファイル転送（SCP）逆順でリモート→ローカルも可
rsync -avz /src/ user@host:/dest/
差分同期（rsync）-n で dry-run（実際は転送しない）
diff -u old.conf new.conf
ファイルの差分確認設定変更前後の比較に
sed -i.bak 's/old/new/g' file.conf
ファイル内の文字列を一括置換.bak でバックアップ自動作成

👤 User / Permission

コマンド用途補足
useradd -m -s /bin/bash USERNAME
ユーザー作成-m でホームディレクトリ作成
usermod -aG wheel USERNAME
sudo グループに追加-aG で既存グループを保持しつつ追加
chmod 644 file / chmod 750 script.sh
パーミッション変更644=rw-r--r-- / 750=rwxr-x---
chown -R user:group /path/
所有者を再帰的に変更-R でディレクトリ以下すべて
id && sudo -l
自分のユーザー情報・sudo 権限確認whoami でユーザー名だけ表示

📈 Performance

コマンド用途補足
vmstat 1 10
CPU / メモリ / I/O の統合確認r:実行待ち si/so:スワップ bi/bo:ディスクI/O
sar -u 1 5
CPU 使用率の推移-r メモリ / -n DEV ネットワーク (sysstat 必要)
ss -s
TCP 接続状況サマリーss -tnp state ESTABLISHED で接続一覧

💡 便利 Tips

コマンド / 設定用途補足
history | grep dnf
コマンド履歴の絞り込み!123 で番号指定再実行 / !! で直前を再実行
grep -v "^#\|^$" /etc/sample.conf
コメント行・空行を除いて表示設定ファイル確認の定番ワンライナー
find / -type f -printf '%s %p\n' 2>/dev/null | sort -rn | head -10
ディスク上の大きいファイル TOP10容量調査に有効
screen -S mysession
セッション維持（切断されても作業継続）Ctrl+a d で切り離し / screen -r で再接続
tmux new -s work
tmux でセッション管理Ctrl+b d で切り離し / tmux attach -t work で再接続
ssh-keygen -t ed25519 -C "mykey"
SSH 鍵ペア生成ssh-copy-id user@host で公開鍵を転送
curl -o /dev/null -s -w "%{http_code}\n" http://example.com
HTTP ステータスコード確認-I でヘッダーのみ / -v で詳細表示
