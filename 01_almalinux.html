1. Packages

パッケージのインストール

bash
dnf -y update && dnf -y install epel-release dnf-utils \
       vim tar tree jq curl git bash-completion python3 ca-certificates \
       tcpdump lsof traceroute bind-utils nmap-ncat net-snmp-utils

SSL エラーが出る場合

bash
# 名前解決の確認
ping www.yahoo.co.jp

# SSL 検証を無効化してインストール
dnf -y update --setopt=sslverify=false && dnf -y install epel-release --setopt=sslverify=false

2. Disable filter

Firewalld を停止・無効化

bash
systemctl stop firewalld.service && systemctl disable firewalld.service

SELinux を無効化

bash
sed -i '/enforcing/s/^/#/g' /etc/selinux/config
sed -i '/#SELINUX=enforcing/a SELINUX=disabled' /etc/selinux/config

設定確認

bash
sestatus
output
SELinux status:                 enabled
SELinuxfs mount:                /sys/fs/selinux
SELinux root directory:         /etc/selinux
Loaded policy name:             targeted
Current mode:                   enforcing
Mode from config file:          disabled           ← 変更されている
Policy MLS status:              enabled
Policy deny_unknown status:     allowed
Memory protection checking:     actual (secure)
Max kernel policy version:      33

設定を適用（再起動）

bash
shutdown -r now
⚠ 注意 — 再起動後に SELinux が Disabled になっていることを sestatus で再確認してください。

3. Network

NetworkManager の状態確認

bash
systemctl status NetworkManager.service --no-pager

物理インターフェース一覧

bash
nmcli d status
output

eth0    ethernet  接続済み  eth0
eth2    ethernet  切断済み  --

bash
nmcli -t d show
